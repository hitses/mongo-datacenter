Introducción a MongoDB

Base de datos de documentos

Un registro en MongoDB es un documento, que es una estructura de datos compuesta por pares de campo y valor. Los documentos de MongoDB son similares a los objetos JSON. Los valores de los campos pueden incluir otros documentos, matrices y matrices de documentos.

Las ventajas del uso de documentos son:

  • Los documentos corresponden a tipos de datos nativos de muchos lenguajes de programación.
  • Los documentos y matrices integrados reducen la necesidad de costosas uniones.
  • El esquema dinámico admite el polimorfismo fluido.

Colecciones/Vistas/Vistas materializadas bajo demanda

MongoDB almacena documentos en colecciones. Las colecciones son análogas a las tablas de las bases de datos relacionales.

Además de colecciones, MongoDB soporta:

  • Vistas de sólo lectura (a partir de MongoDB 3.4)
  • Vistas materializadas bajo demanda (a partir de MongoDB 4.2).

Características principales

Alto rendimiento

MongoDB proporciona una persistencia de datos de alto rendimiento. En particular,

  • La compatibilidad con modelos de datos incrustados reduce la actividad de E/S en el sistema de base de datos.
  • Los índices permiten consultas más rápidas y pueden incluir claves de documentos y matrices incrustados.

API de consulta

La API de consulta de MongoDB admite operaciones de lectura y escritura (CRUD), así como:

  • Agregación de datos
  • Búsqueda de texto y consultas geoespaciales.

Alta disponibilidad

La facilidad de replicación de MongoDB, llamada conjunto de réplicas, proporciona:

  • conmutación automática por error
  • redundancia de datos.

Un conjunto de réplicas es un grupo de servidores MongoDB que mantienen el mismo conjunto de datos, proporcionando redundancia y aumentando la disponibilidad de los datos.

Escalabilidad horizontal

MongoDB proporciona escalabilidad horizontal como parte de su funcionalidad principal:

  • La fragmentación distribuye los datos a través de un cluster de máquinas.
  • A partir de la versión 3.4, MongoDB soporta la creación de almacenamiento por zonas de datos basadas en la clave de fragmentación. En un cluster equilibrado, MongoDB dirige las lecturas y escrituras cubiertas por una zona sólo a aquellos fragmentos dentro de la zona.

Soporte para múltiples motores de almacenamiento

MongoDB soporta múltiples motores de almacenamiento:

  • Motor de almacenamiento WiredTiger (incluyendo soporte para encriptación en reposo).
  • Motor de almacenamiento en memoria.

Además, MongoDB proporciona una API de motor de almacenamiento conectable que permite a terceros desarrollar motores de almacenamiento para MongoDB.
